Reports cases when non-safe string is passed to a method with parameter marked with @Untainted annotations.

Safe string is:

  • call of method that is marked as @Untainted
  • local variable or method parameter that does not call non-safe methods
  • field, local variable or parameter that is marked as @Untainted and does not have non-safe methods calls assigned

Example: 


  void doSmth(boolean b) {
    String s = safe();
    String s1 = "other";
    if (b) s1 = s;
    sink(s);
  }
  
  String sink(@Untainted String s) {}

Here we do not have non-safe string assignments to s so warning is not produced. On the other hand: 


  void doSmth(boolean b) {
    String s = safe();
    String s1 = "other";
    s1 = foo();
    if (b) s = s1;
    sink(s);        // warning here
  }
  
  String foo();

  String sink(@Untainted String s) {}

Here we have a warning since s1 has an unknown state after foo call result assignment.

  • Use the Untainted annotations options to specify annotations, which will be used as @Untainted annotations during the analysis. The first annotation from this list will be used for propagation if it exists in classpath.

  • Use the Tainted annotations options to specify annotations, which will be used as @Tainted annotations during the analysis

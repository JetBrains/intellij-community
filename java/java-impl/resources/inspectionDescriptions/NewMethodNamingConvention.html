Reports methods whose names are too short, too long, or do not follow the specified regular expression pattern.

A quick-fix to rename such methods is available only in the editor.

Instance methods that override library methods and constructors are ignored by this inspection.

Example:


  public static int n() {
    return 0;
  }
This example will be reported if the inspection is enabled for static methods with the default settings where a method name should have a minimum length of 4.

Use the list in the Options section to specify the types of methods that should be checked.

For each method type, you can specify the minimum length, maximum length, and regular expression for acceptable method names. Specify 0 in a length field to skip the corresponding check. Regular expressions use the standard java.util.regex format.

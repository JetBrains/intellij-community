Reports any calls to Math.random() which are immediately cast to int. Casting a double between 0.0 (inclusive) and 1.0 (exclusive) will always round down to zero. A Math.random() value should first be multiplied with some factor before casting it to an int to get a value between zero (inclusive) and the multiplication factor (exclusive). Another possible solution would be to use the nextInt() method of java.util.Random.

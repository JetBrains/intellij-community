Reports method parameters that are named identically to the visible fields of the method's class or surrounding classes if it is a parameter inside of an inner or anonymous class. Such a parameter name may be confusing and it's possible to accidentally use the parameter where the field is intended.

The quick-fix is suggested to rename the parameter.

Example:

  
    class Main {
      private final int value;

      public Main(int value) {
          this.value = value;
      }
    }

Use the "Options" panel to configure the inspection:

  1. Enable the "Ignore for property setters" checkbox to ignore parameters of simple setters.
  2. Enable the "Ignore superclass fields not visible from subclass" checkbox to ignore private fields in a superclass, which are not visible from the parameter.
  3. Enable the "Ignore for constructors" checkbox to ignore parameters of constructors.
  4. Enable the "Ignore for abstract methods" checkbox to ignore parameters of abstract methods.
  5. Enable the "Ignore for static method parameters hiding instance fields" checkbox to ignore parameters of static methods hiding an instance field and to ignore parameters of instance methods in static inner classes hiding an instance field in an outer class. While not strictly hiding, such parameters can still be confusing.

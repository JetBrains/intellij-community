Reports JUnit 4 parameterized test classes annotated with @RunWith(Parameterized.class) without a data provider method annotated with @Parameterized.Parameters or with a data provider method that has an incorrect signature. A data provider method should be public, static and have a return type of Iterable or Object[]. Such test classes cannot be run.

For example:


  @RunWith(Parameterized.class)
  public class ImportantTest {
    private int input;
    private int expected;

    ImportantTest(int input, int expected) {
      this.input = input;
      this.expected = expected;
    }

    // ... test cases
  }

A quick fix is provided to create an empty parameter provider method (though you'll need to implement it manually):


  @RunWith(Parameterized.class)
  public class ImportantTest {
    private int input;
    private int expected;

    ImportantTest(int input, int expected) {
      this.input = input;
      this.expected = expected;
    }

    @Parameters
    public static Iterable<Object[]> parameters() {
      return null;
    }

    // ... test cases
  }

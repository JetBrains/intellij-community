This inspection reports exceptions constructed and thrown from inside catch blocks, which do not "wrap" the caught exception. It is considered good practice when throwing an exception in response to an exception to wrap the initial exception, so that valuable context information such as stack frames and line numbers are not lost.

Use the checkbox below to indicate if the inspection should ignore exceptions which receive the result of a method call on the original exception, such as getMessage(), as an argument.

Reports SLF4J or Log4j 2 logging calls, such as logger.info("{}: {}", key) where the number of {} placeholders in the logger message doesn't match the number of other arguments to the logging call.

Use the inspection option, if you want to print a stack trace for last arguments with an exception type. This option works only for SLF4J 


  log.error("Error id {} for task {}", id, e); // will be reported

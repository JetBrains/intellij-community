Reports generic method parameters which can use bounded wildcards.
Tries to enforce a widely accepted principle of API design: use wildcards in the method signature.

To quote Josh Bloch in "Effective Java" third Edition item 31:

Use bounded wildcards to increase API flexibility.

Using wildcard types in your APIs, while tricky, makes the APIs far more flexible. Remember the basic rule: producer-extends, consumer-super (PECS). And remember that all Comparables and Comparators are consumers.

Example: 

<T> void process(Consumer<T> consumer);
should be replaced with: 
<T> void process(Consumer<? super T> consumer);
 This method signature is more flexible because it accepts more types.

Likewise, type parameters in covariant position: 

<T> T produce(Producer<T> p);
should be replaced with: 
<T> T produce(Producer<? extends T> p);

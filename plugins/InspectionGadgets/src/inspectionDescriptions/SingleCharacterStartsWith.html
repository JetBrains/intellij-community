Reports any calls to String.startsWith() or String.endsWith() where single character string literals passed as a parameter.

There is a fix that replaces such calls with more efficiently implemented String.charAt().

Because the performance gain is minimal and the code becomes less readable because of the extra non-zero length check, it is recommended to do so only inside tight loops.

This inspection is intended for Java ME and other highly resource constrained environments. Applying the results of this inspection without consideration might have negative effects on code clarity and design.

Example:

  boolean startsWithX(String s) {
    return s.startsWith("x");
  }

After the quick-fix is applied:

  boolean startsWithX(String s) {
    return !s.isEmpty() && s.charAt(0) == 'x';
  }

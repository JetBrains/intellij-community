Reports problems in Comparator.compare() implementations.

The following cases are reported:

  • When a parameter is not used. Most likely this is a typo and the other parameter is compared with itself, or the method is not implemented correctly.
  • When it's evident that the comparator does not return 0 for the same elements. Such comparator violates the contract and can produce unpredictable results when equal elements are encountered. In particular, sorting may fail with an exception on some data sets.

Example:

  Comparator<String> lambda = (a, b) -> (a.length() > b.length() ? 0 :
                Math.random() > 0.5 ? (-1) : (1));

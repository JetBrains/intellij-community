Reports any methods which override methods in a superclass but do not have the @java.lang.Override annotation.

Having annotation is good for readability because it shows intent. Also in this case compiler emits error when signature of the overridden method doesn't match super class method.

Configure the inspection:

  • Use the first checkbox below to have this inspection ignore the java.lang.Object methods equals(), hashCode() and toString(). The risk that these methods will disappear and thus your code won't be compiling anymore due to the @Override annotation is vanishingly small.
  • Use the second checkbox below to have this inspection ignore overriding methods in anonymous classes. It might be useful for you if you want anonymous classes to be as short as possible.

This inspection only reports if the language level of the project or module is 5 or higher

Reports array fields which are declared as volatile. Such fields may be confusing, as accessing the array itself follows the rules for volatile fields, but accessing the array's contents does not. If such volatile access is needed to array contents, java.util.concurrent.atomic classes should be used instead.

Example:

class Data {
  private volatile int[] idx = new int[0];
}
Good code which allows volatile array content access should look like private final AtomicIntegerArray idx = new AtomicIntegerArray(new int[0]);

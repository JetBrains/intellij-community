Reports on any call to java.util.concurrent.locks.Condition.await() not made inside a loop. await() and related methods are normally used to suspend a thread until some condition is true. Because the thread could have been waken up for a different reason, the condition should be checked after the await() call returns. A loop is a simple way to achieve this.

Example:

void acquire(Condition released) throws InterruptedException { 
  released.await();
}

Good code should look like this: 

void acquire(Condition released) throws InterruptedException {
  while (acquired) {
    released.await();
  }
}

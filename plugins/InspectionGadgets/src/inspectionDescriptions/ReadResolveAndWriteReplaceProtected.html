Reports classes which implement java.io.Serializable interface where the readResolve() or writeReplace() methods are not declared protected. Declaring readResolve() and writeReplace() methods private can force subclasses to silently ignore them, while declaring them public allows them to be invoked by untrusted code. Note: in the case of classes declared final, these methods may be declared private, instead.

Example:


  class ClassWithSerialization implements Serializable {
    public Object writeReplace() { // warning: 'writeReplace()' not declared protected
        ...
    }
  }

Reports methods with too many return points. Methods with too many return points may be confusing, and hard to refactor. A return point is either a return statement or the falling through the bottom of a void method or constructor.

For example, such method is reported if inspection allows only two return statement:


  void doSmth(User[] users) {
    for (User user : users) {
      if (cond1(user)) {
        user.setId(getId());
        return;
      } else if (cond2(user)) {
        if (cond3(user)) {
          user.setId(getId());
          return;
        }
      }
    }
  }

And it could be rewritten in an easier to understand way:


  void doSmth(User[] users) {
    for (User user : users) {
      if (cond1(user) || cond2(user) && cond3(user)) {
        user.setId(getId());
        return;
      }
    }
  }

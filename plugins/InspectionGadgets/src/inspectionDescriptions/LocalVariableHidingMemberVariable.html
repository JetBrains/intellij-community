Reports local variables named identically to fields of their class or surrounding classes if it is a local variable inside of an inner or anonymous class. Such a variable name may be confusing.

Use the quick-fix to rename the local variable.

Example:


  public class Foo {
    public Object foo;

    void bar() {
      Object o = new Object() {
        void baz() {
          Object foo; // Local variable 'foo' hides field in class 'Foo'
        }
      };
    }
  }

Use the first checkbox below to ignore local variables hiding a super class field which is not visible (e.g. because it's private).

Use the second checkbox below to ignore local variables in a static context, named identically to a non-static field. For example when the local variable is inside a static method or inside a method which is inside an static inner class.

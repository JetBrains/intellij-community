Reports the @DependsOn, @Repository, and @Import annotations when they are used in a plain Kotlin script (.kts) rather than a main Kotlin script (.main.kts).

These annotations are contributed only by the main Kotlin Scripting definition, so they are supported exclusively in .main.kts files. A plain .kts script has no functionality that processes them: the references do not resolve, and the dependencies or script imports they would declare never take effect.

The quick fix renames the file to .main.kts, which switches the script definition so that the annotations resolve and work as intended. 


@file:DependsOn("com.example:my-library:1.0")
@file:Import("helper.main.kts")

// In a plain .kts script these annotations are unresolved and have no effect.
println("Hello")

If you do not actually need external dependencies, you can keep the file as a plain .kts script and remove the annotations instead.

Reports property names that do not follow the recommended naming conventions.

Consistent naming allows for easier code reading and understanding. According to the Kotlin official style guide, property names should start with a lowercase letter and use camel case.

It is possible to introduce other naming rules by changing the "Pattern" regular expression.

Example:


  val My_Cool_Property = ""

The quick-fix renames the class according to the Kotlin naming conventions:


  val myCoolProperty = ""

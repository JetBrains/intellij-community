Reports not-null assertion (!!) calls that can be replaced with the elvis operator and error(...).

A not-null assertion can lead to an unexpected NPE (NullPointerException). Replacing it with ?: error(...) makes the failure explicit and supports whole-file inspection cleanup.

Example:


fun test(number: Int?) {
    val x = number!!
}

After the quick-fix is applied:


fun test(number: Int?) {
    val x = number ?: error("Expression 'number' must not be null")
}

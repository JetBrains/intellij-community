Reports duplicate constant elements in set-like collection factory functions and duplicate constant keys in map-like collection factory functions.

Duplicate entries in collection factory functions are usually unintended and may indicate a bug or typo. In set-like functions, duplicate elements are ignored because sets contain unique elements only. In map-like functions, duplicate keys overwrite previous values, and only the last value associated with the key is retained.

The inspection applies to the following map-like functions:

  • mapOf()
  • mutableMapOf()
  • hashMapOf()
  • linkedMapOf()
  • sortedMapOf()

The inspection applies to the following set-like functions:

  • setOf()
  • mutableSetOf()
  • hashSetOf()
  • linkedSetOf()
  • sortedSetOf()

The inspection reports duplicates for compile-time constant values such as:

  • String
  • Char
  • Numeric types (Int, Long, Short, Byte, Float, Double)
  • const val references

Examples:


val numbers = setOf(1, 1, 2)

The second 1 is redundant because sets cannot contain duplicate elements. 


val mapping = mapOf(
    1 to "first",
    1 to "second",
)

The second entry overwrites the first one because map keys must be unique.

Example with const val:


const val ID = 42

val ids = setOf(ID, 42)

Both arguments resolve to the same constant value.

The quick-fix suggests removing redundant entries or duplicate keys.

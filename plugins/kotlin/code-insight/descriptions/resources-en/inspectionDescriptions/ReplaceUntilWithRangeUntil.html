Reports until that can be replaced with ..< operator.

Every until to ..< replacement doesn't change the semantic in any way.

The UX research shows that developers make ~20-30% fewer errors when reading code containing ..< compared to until.

Example:


  fun main(args: Array<String>) {
      for (index in 0 until args.size) {
          println(index)
      }
  }

After the quick-fix is applied:


  fun main(args: Array<String>) {
      for (index in 0..<args.size) {
          println(index)
      }
  }

This inspection only reports if the Kotlin language level of the project or module is 1.7 or higher, and -opt-in=kotlin.ExperimentalStdlibApicompiler flag is provided.

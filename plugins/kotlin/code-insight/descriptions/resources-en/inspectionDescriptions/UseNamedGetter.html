Reports calls to generated componentN() methods on Kotlin data classes from Java code.

Kotlin data classes automatically generate positional component1(), component2(), and similar methods for destructuring. Calling them directly from Java is fragile: the mapping between index and property is not obvious, and the calls break silently if properties are reordered. Using the named property getter is clearer and refactoring-safe.

Example:


  String name = person.component1();
  int age = person.component2();

After the quick-fix is applied:


  String name = person.getName();
  int age = person.getAge();

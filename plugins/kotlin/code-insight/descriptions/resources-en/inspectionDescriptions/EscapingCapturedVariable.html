Reports mutable local variables (var) captured by an escaping lambda, that is, a lambda that may run after the enclosing function has finished, such as a lambda passed to lazy, launch, or similar.

Lambdas capture local variables by reference, so if the variable is reassigned before the lambda runs, the lambda observes the new value, not the value from the moment of capture. Because the capture and the reassignment are usually far apart in the code, this leads to bugs that are hard to find.

Example:


  var seed = 3
  val deferred = lazy { seed * 2 } // 'seed' is captured here
  seed = 4
  println(deferred.value) // prints 8, not 6

To make the timing explicit, create a new val with a snapshot


  var seed = 3
  val snapshot = seed
  val deferred = lazy { snapshot * 2 }
  seed = 4
  println(deferred.value) // prints 6

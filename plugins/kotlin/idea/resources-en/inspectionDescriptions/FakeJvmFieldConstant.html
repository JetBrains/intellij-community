Reports non-const Kotlin properties used as Java annotation arguments. For example a property which is annotated with @JvmField, has an initializer that can be evaluated at compile-time, has a primitive or String type.

Such properties have a ConstantValue attribute in bytecode in Kotlin 1.1-1.2. This attribute allows javac to fold usages of the corresponding field and use that field in annotations. This can cause incorrect behavior in case of separate or incremental compilation in mixed Java/Kotlin code. This behavior is subject to change in Kotlin 1.3 (no ConstantValue attribute any more). Recommended action: replace the @JvmField annotation with a const modifier on a relevant Kotlin property or inline it.

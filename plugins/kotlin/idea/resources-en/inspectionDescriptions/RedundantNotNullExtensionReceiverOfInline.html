Reports inline functions with not nullable extension receivers which does not use the fact that extension receiver is not nullable.

In Kotlin 1.0 or 1.1, calls of inline fun with flexible nullable extension (platform type with unknown nullability) receiver did not include nullability check in bytecode, but in Kotlin 1.2 this problem is fixed (see KT-12899), nullability check is included into bytecode.

Thus functions which do not use the fact that extension receiver is not nullable are dangerous in Kotlin until 1.2 and it's recommended to make such functions to have nullable receiver.

This inspection only reports if the Kotlin language level of the project or module is lower than 1.2.

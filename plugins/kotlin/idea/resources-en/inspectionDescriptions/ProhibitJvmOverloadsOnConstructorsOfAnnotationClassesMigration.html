Reports @JvmOverloads on constructors of annotation classes cause it's meaningless.

There is no footprint of @JvmOverloads in the generated bytecode and Kotlin metadata, so @JvmOverloads doesn't affect generated bytecode and behavior of the code.

@JvmOverloads on constructors of annotation classes is warning since Kotlin 1.3 and compilation error since Kotlin 1.4.

Example:


  annotation class A @JvmOverloads constructor(val x: Int = 1)

After the quick-fix is applied:


  annotation class A constructor(val x: Int = 1)

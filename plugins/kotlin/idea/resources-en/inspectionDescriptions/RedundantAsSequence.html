Reports redundant asSequence() call that can never have a positive performance effect.

Normally, asSequence speeds up collection processing that includes multiple operations by performing them lazily. However, if a terminal operation (such as toList()) is used right after asSequence(), this doesn't give you any positive performance effect.

Example:


fun test(list: List<String>) {
    list.asSequence().last()
}

After the quick-fix is applied:


fun test(list: List<String>) {
    list.last()
}

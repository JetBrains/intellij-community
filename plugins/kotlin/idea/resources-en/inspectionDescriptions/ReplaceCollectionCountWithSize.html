Reports usages of collection Collection<T>.count(). This function call can be replaced with .size.

.size form emphasizes that the operation is O(1) and won't allocate whereas count() could be confused with Iterable<T>.count() which is O(n) and allocating.
"Replace 'count' with 'size'" quick-fix allows to amend the highlighted code snippet.

Example:


   fun foo() {
      var list = listOf(1,2,3)
      list.count() // 'list.size' could be used instead
   }
